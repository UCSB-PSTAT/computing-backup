Why use developer containers?

Developer containers allow developers to package their application along with all the necessary dependencies, libraries, and configurations, into a single lightweight image that can be easily deployed and run across different environments. This eliminates the common issue of "it works on my machine" and ensures that the application runs the same way in any environment. Furthermore, using containers can simplify the development process and increase productivity, as users can easily switch between different environments (whether locally on a laptop or remotely on a supercomputing cluster) without worrying about installing and managing dependencies manually.

This is all done using a simple 2 file setup and a tool called "Podman" (Pod manager). In this tutorial we will be going over:

  • Installing Podman
  • Setting up VS Code
  • Setting up your first dev container
  • Attaching/Detaching to a container

Prereqs

  • (Optional) Setup of the UCSB VPN for connecting to PSTAT servers
